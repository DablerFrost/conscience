# VYRION Daily Operations — Templates
## Last Updated: April 11, 2026

---

## The Daily Record — ATLAS

The VYRION Index is updated daily. The authoritative daily record lives at:

**vyrion.earth/atlas.html**

ATLAS is the product. It is not a social media summary. It is not a scorecard image. It is a live planetary intelligence dashboard that updates automatically via GitHub Actions. If the data is current and the site is live, the daily record exists. That is sufficient.

Social media does not constitute a daily record. Social media is a separate broadcast channel used selectively for featured content. The two are architecturally distinct.

---

## Daily Operations Checklist (5 minutes)

1. Verify ATLAS is loading and data is current at vyrion.earth/atlas.html
2. Verify SHIELD is loading at vyrion.earth/shield.html
3. Check GitHub Actions — confirm conscience-daily.yml and shield-update.yml ran successfully
4. If Index movement exceeds ±0.5 (minimum detectable change), note it for potential social post
5. Commit archive entry to /archive if a significant event warrants logging

No social post is required on any given day. ATLAS handles the daily record automatically.

---

## Archive Entry — When to Log (5 minutes)

Log to /archive when:
- Index moves ±0.5 or more
- A planetary boundary event occurs (significant seismic, wildfire, ice record, etc.)
- A Record issue is published
- A correction is issued under the Open Proof Protocol
- An institutional milestone occurs (outreach sent, response received, partnership)

Copy TEMPLATE-daily.md → rename to YYYY-MM-DD.md → fill 4 fields → commit to /archive.

Do not log every day for the sake of logging. The archive is a meaningful record, not a diary.

## Social Media — Featured Broadcast Channel

Social media (X @DablerFrost · Instagram @DablerFrost · LinkedIn) is used for featured content only. It is not part of VYRION's architecture. It does not constitute a record. It is a broadcast tool used when there is something worth amplifying.

### When to post:

**Weekly — The Record**
Every Wednesday when a Record issue publishes. Screenshot the issue HTML. Caption includes: pledge vs reality summary, CONSCIENCE score, source citation, link to vyrion.earth.

**As needed — Major Index Movements**
When the VYRION Index moves by a meaningful amount (±1.0 or more, or a sustained directional trend). Frame the movement with one data source. Link to ATLAS.

**As needed — Institutional Announcements**
Platform milestones, methodology updates, new modules, partnerships, corrections under Open Proof Protocol.

**As needed — Starbucks and Company Posts**
When a company's CONSCIENCE score update or filing warrants public commentary. Requires CONSCIENCE-METHODOLOGY.md breakdown confirmed first.

### What social media is NOT used for:
- Daily Index score cards
- Routine data updates
- Replacing ATLAS as the record
- Any content that does not add meaningful context beyond what the platform already shows

## Open Proof Protocol — Correction Procedure

When an error is identified in any published post, platform, or document:

1. Post public correction as reply to original — same day
2. State exactly what was wrong and what the correct information is
3. Cite the correct source
4. Update GitHub if it affects methodology or platform data
5. Never delete originals — corrections are additive and public

Example (Day 2 Arctic correction, March 20, 2026):
"Correction to Day 2: Arctic sea ice extent of 14.22M km² represents the 2026 annual maximum — among the lowest March peaks in 47 years of satellite records. Minimums occur in September. Source: NSIDC. VYRION publishes corrections openly. #VYRIONIndex"

## Index Reporting Format — When Referenced

Always report the VYRION Index as:

**XX.X ± 1.8 / 100**

Day-to-day changes under ±0.5 are within measurement error. Do not describe sub-threshold movements as significant in any published material.
